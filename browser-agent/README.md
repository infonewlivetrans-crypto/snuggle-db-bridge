# Radius Track Browser Agent (dev)

Дев-версия MV3-расширения Chrome, которое помогает диспетчеру Радиус Трек
работать поверх открытого сайта ATI в его собственном браузере.

**Версия:** 0.2.0
**Протокол:** 1
**Selector config:** dev-1
**Канал:** dev

## 1. Что это

Radius Track Browser Agent — вспомогательное расширение. Оно НЕ является
клиентом ATI и НЕ использует API ATI. Оно только:

- открывает страницу ATI в вашей же вкладке;
- читает видимую пользователю выдачу грузов;
- отправляет найденные грузы на бэкенд Радиус Трек по вашему `agent_token`.

## 2. Что расширение делает

- открывает нужную страницу ATI;
- заполняет форму поиска, если Радиус Трек прислал команду;
- читает список видимых грузов;
- отправляет их на `/api/public/agent/ai-dispatcher/loads`;
- подсвечивает подходящие грузы прямо на странице ATI;
- фокусирует конкретный груз по запросу диспетчера;
- присылает heartbeat раз в 30 секунд;
- показывает состояние в popup.

## 3. Что расширение НЕ делает

- НЕ вызывает API ATI;
- НЕ читает cookies ATI;
- НЕ читает `localStorage` ATI;
- НЕ хранит логин и пароль ATI (их вводите только вы, на самой ATI);
- НЕ передаёт наружу номера телефонов, email и контакты;
- НЕ пишет сырой HTML страницы в диагностику;
- НЕ показывает `agent_token` после подключения.

## 4. Требования

- Chrome / Edge / Brave / Arc / Opera (Chromium 110+);
- активный аккаунт диспетчера в Радиус Трек;
- pairing-код из кабинета AI-диспетчера.

## 5. Сборка

```bash
cd browser-agent
npm install
npm run typecheck
npm run build     # → dist/
npm test          # node --test tests/*.test.mjs
npm run package   # → packaged/radius-track-agent-0.2.0.zip (если есть zip)
```

## 6. Установка в Chrome

1. Выполнить `npm run build`.
2. Открыть `chrome://extensions`.
3. Включить **Режим разработчика** (переключатель справа сверху).
4. Нажать **Загрузить распакованное расширение**.
5. Выбрать папку `browser-agent/dist`.

Работает во всех Chromium-браузерах (Chrome, Edge, Brave, Arc, Opera).

## 7. Подключение по pairing-коду

1. В Радиус Трек: страница AI-диспетчера → «Подключить Browser Agent» →
   получить код вида `RT-XXXX-XXXX`.
2. В popup расширения указать URL Радиус Трек (например,
   `https://your-project.lovable.app`).
3. Ввести pairing-код, нажать **Подключить**.
4. Popup покажет **Paired · токен активен**.

Agent token хранится только в `chrome.storage.local` и после подключения
пользователю больше не показывается.

## 8. Настройка адреса Радиус Трек

- Поле «URL Радиус Трек» в popup.
- Допустимые схемы: только `https://` (или `http://localhost` для dev).
- Схемы `javascript:`, `file:`, `data:` заблокированы.

## 9. Проверка соединения

Кнопка **Проверить соединение** делает `GET /api/public/agent/ai-dispatcher/health`.
Ответ содержит только whitelist:

- `status`
- `current_web_version`
- `agent_protocol_version`
- `minimum_agent_version`
- `recommended_agent_version`
- `supported_selector_config_versions`
- `server_time`

Никаких session id, user id, token, pairing code, задач и кандидатов.

## 10. Запуск поиска

- В Радиус Трек создать задачу поиска в AI-диспетчере.
- Нажать «Начать поиск» — Радиус Трек отправит расширению команду,
  расширение откроет ATI и заполнит форму.
- В popup будет отображаться «Задача», «Видно», «Отправлено», «Подходит».

## 11. Работа с несколькими автомобилями

- Радиус Трек умеет запускать несколько параллельных задач поиска.
- Расширение открывает по одной вкладке на задачу.
- Закрывает только те вкладки, которые открывало само.

## 12. Диагностика

Кнопка **Открыть диагностику** возвращает безопасный JSON:

- версия агента, протокола, selector config;
- build channel и дата сборки;
- URL страницы без query;
- тип страницы;
- количество найденных inputs и строк грузов;
- какие поля формы найдены/пропущены;
- статус последней команды;
- количество активных вкладок.

Не содержит: `agent_token`, `token_hash`, `pairing_code`, cookies,
`localStorage` ATI, логин, пароль, телефон, email, полный HTML.

## 13. Обновление расширения

1. `git pull` (или скачать новую версию).
2. `npm run build`.
3. В `chrome://extensions` нажать **Обновить** на карточке расширения.
4. Popup покажет новую версию и обновит совместимость.

Расширение автоматически проверяет совместимость версии Web ↔ Agent.
См. AgentHealthPanel в кабинете диспетчера.

## 14. Что делать при несовместимой версии

Возможные статусы совместимости:

- **compatible** — всё в порядке.
- **update_recommended** — работает, но рекомендуется обновиться.
- **unsupported** — версия ниже минимальной; обновите расширение.
- **protocol_mismatch** — протокол несовместим; обновите расширение
  или Радиус Трек.
- **selector_config_warning** — неизвестная версия selector config;
  часть автоматических действий может не работать.

Расширение не отзывает токен и не удаляет сессию автоматически.
При `unsupported` / `protocol_mismatch` — обновитесь по инструкции выше.

## 15. Что делать, если ATI изменил разметку

- Диагностика покажет `search_form_not_detected` или `extraction_failed`.
- Расширение НЕ будет считать все грузы пропавшими (см.
  `MissingCandidatesPanel` в Радиус Трек).
- Скопируйте безопасную диагностику и отправьте её команде разработки.
- Обновите `selector_config_version` в следующей версии агента.

## 16. Безопасность

- Расширение общается только с бэкендом Радиус Трек.
- Bearer `agent_token` хранится в `chrome.storage.local`.
- `service_role` бэкенда не используется, RLS не отключается.
- Публичный health не отдаёт персональные данные.
- Персональный `session-health` защищён Bearer `agent_token`
  и возвращает только вашу сессию.
- Логин и пароль ATI вводит только пользователь, на самой странице ATI.
- Cookies ATI расширение не читает.

## 17. Удаление / отключение

- В `chrome://extensions` — переключатель «Включено» или кнопка
  **Удалить**.
- В popup — кнопка **Отключить** очищает `agent_token` и `session_id`
  из локального хранилища.
